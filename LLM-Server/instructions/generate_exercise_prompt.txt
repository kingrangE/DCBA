# Task
전달받은 [과목명]과 [난이도]에 근거하여, 실제 대기업 기술 면접에서 나올 법한 날카로운 면접 질문과 그에 대한 모범 답안을 생성하라.

# Input Data
- 과목명: {subject}
- 난이도: {level}

# Complexity Guidelines
- Level 1 (기초): 핵심 개념의 정의와 사용 목적 (예: "왜 이 기술이 탄생했는가?")
- Level 2 (기본): 내부 동작 원리 및 장단점 비교 (예: "내부적으로 어떤 메커니즘으로 동작하며, 어떤 한계가 있는가?")
- Level 3 (심화): 복잡한 시스템 설계 및 트러블슈팅 (예: "특정 장애 상황이나 대용량 환경에서 이 기술을 어떻게 최적화할 것인가?")

# Output Format (Strictly Enforced)
반드시 아래의 2줄 형식을 유지하라. Q와 A 사이의 개행은 딱 1번만 허용하며, 답변(A) 내부에서도 줄바꿈을 하지 않는다.

Q: [면접관이 질문하듯 자연스러운 구어체로 작성]
A: [지원자가 답변하듯 논리적이고 전문적인 구어체로 작성. 서론-본론-결론의 구조를 갖추되 시니어급 식견이 드러나도록 함]

---
# Example Output (이 톤을 유지할 것)
Q: "Redis를 캐시로 사용할 때 발생할 수 있는 Cache Stampede 현상이 무엇인지 설명해주시고, 이를 방지하기 위한 본인만의 전략이 있다면 말씀해주세요."
A: "네, Cache Stampede는 대규모 트래픽 환경에서 캐시가 만료되는 순간 여러 요청이 한꺼번에 DB로 몰려 시스템 부하가 급증하는 현상입니다. 저는 이를 방지하기 위해 단순히 TTL을 고정하지 않고 만료 시간에 '지터(Jitter)'를 추가하거나, 확률적 만료 기법을 도입하여 요청을 분산시키는 방식을 선호합니다."

# Formatting Rules
1. Q와 A 사이를 제외한 곳에 빈 줄(Empty line)을 넣지 말 것.
2. 답변이 길어지더라도 한 줄에 모두 작성할 것(No internal line breaks).
